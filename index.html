Hardware-Enabled Root of Trust

A Root of Trust (RoT) is a source that is trusted by default within a cryptographic system. Traditionally, software roots of trust are vulnerable because if a hacker obtains kernel/root access, they can alter the trust verification code. Hardware-Enabled Root of Trust anchors security in immutable hardware modules (like the Trusted Platform Module / TPM or Secure Enclave). It is physically isolated from the CPU, containing its own cryptographic engines, non-volatile memory, and unique factory-burned endorsement keys.

Hardware-Backed Identity

WebAuthn allows websites to leverage local hardware authenticators (TPMs, Windows Hello, YubiKeys) for secure, passwordless authentication. The private keys never leave the hardware boundary. During register/login, the hardware generates a signature using these keys. Our demo shows the actual cryptographic assertions and verifies them in real time.

Boot Integrity & Storage

Measured Boot computes cryptographic hashes of every step of the boot loader chain, compounding them into the hardware TPM's Platform Configuration Registers (PCRs). Remote Attestation sends signed PCR reports to a verifier (Firebase) to check for malware tampering. Sealed Storage encrypts data such that it only decrypts if the boot state matches reference values.

TPM Platform Configuration Registers (PCRs)

Platform Configuration Registers (PCRs) are cryptographic registers inside the hardware TPM. They cannot be written directly; they can only be extended: PCR_val = SHA256(PCR_val + Measurement). This creates a cryptographically un-forgeable audit trail of the boot sequence.

Seal Data to Platform Config

Sealing binds secret data (like disk encryption keys or API tokens) to specific TPM PCR values. The hardware TPM will only release (unseal) this data if the system's current PCR hashes exactly match the configuration selected at sealing time.
